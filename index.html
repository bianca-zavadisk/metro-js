01 — Abstract

Project Overview

What MetroJS does, why it matters, and how it approaches mesh quality evaluation.

🎯

Objective

Develop a browser-based tool to evaluate the 3D geometric fidelity of simplified meshes against their high-resolution originals, computing both standard numerical metrics and signed distance error maps.

🔬

Method

Stochastic surface sampling via area-weighted Monte Carlo. For each sample on the reference surface, the closest point on the approximation is found using a Bounding Volume Hierarchy, and the signed distance encodes geometric expansion or contraction.

📊

Output

Hausdorff distance, RMS error, mean signed error, and a per-vertex diverging heatmap rendered on the mesh surface. Topology validation reports manifold status, Euler characteristic, and connected components.

E(A, B) = maxp ∈ A   minq ∈ B   ‖ p − q ‖
One-sided Hausdorff distance from mesh A to mesh B
The Metro algorithm approximates this with Monte Carlo sampling: sample points on A, query nearest point on B, record the signed distance.
Full Abstract

The project proposes the development of an interactive web application for the geometric and topological inspection of triangular meshes, inspired by the fundamental concepts of the Metro tool. The primary objective is to evaluate the 3D fidelity of simplified meshes against their high-resolution originals, employing a stochastic surface sampling approach based on the Monte Carlo method. In addition to extracting standard numerical metrics, the application validates mesh structural integrity — identifying connected components and manifold properties — and calculates signed distance to visually distinguish whether the approximation resulted in geometric expansion (positive error) or contraction (negative error). The evaluation results are rendered as a heatmap for visualization.

The implementation is structured using JavaScript for evaluation and topological calculations, and HTML Canvas for graphical rendering and visualization of the computed elements. The entire application runs in the browser without any server dependency, making it portable and accessible for academic demonstration.

🌲 Bounding Volume Hierarchy (BVH)

Brute-force nearest-point queries scale as O(N·M), which is prohibitive for large meshes. MetroJS builds an axis-aligned bounding box (AABB) tree over the target mesh faces, recursively splitting on the longest axis at the median centroid. This reduces nearest-point complexity to O(log N) per sample.

T(N) = O(N log N) build + O(S log N) query

where N = target faces, S = number of MC samples.

🎲 Area-Weighted Monte Carlo Sampling

To ensure uniform coverage over the mesh surface, faces are selected with probability proportional to their area. A uniform random point within each selected triangle is obtained using barycentric coordinates with the square-root method to avoid clustering near the origin vertex.

P(fi) = Area(fi) / Σ Area(fj)

± Signed Distance Computation

Unsigned distance is insufficient for understanding the nature of the approximation error. MetroJS assigns a sign by comparing the vector from the closest point on B to the sample on A against the face normal at the closest point. Alignment → positive (surface expanded); anti-alignment → negative (surface contracted).

sign = sgn( (p − q) · n̂q )

🔺 Closest Point on Triangle

For each sample point, the exact closest point on a target triangle is computed analytically using the Voronoi region decomposition (Eberly 2003). The 7 regions of a triangle (3 vertices, 3 edges, 1 interior) are tested in sequence, yielding the globally closest point without numerical iteration.

q* = argminq ∈ △ ‖p − q‖²
dRMS = √( 1/S · Σi=1S ‖ pi − qi* ‖² )
Root Mean Square error over S Monte Carlo samples. Unlike Hausdorff, RMS is robust to outliers and more representative of average mesh quality.

04 — Topology

Data Structures & Topological Operators

The mathematical foundation for representing and validating triangular meshes as topological surfaces.

⬡ Half-Edge / Edge Map

MetroJS uses a directed edge map (each undirected edge stored once with a canonical key min(a,b)_max(a,b)) to enumerate all edges and their adjacent faces. This allows O(1) lookup for any edge's neighbourhood.

A triangular mesh is manifold if and only if every edge is adjacent to exactly one (boundary) or two (interior) faces. Edges with three or more adjacent faces are non-manifold and indicate invalid geometry.

Edge adjacency: O(F) time, O(E) space

χ Euler Characteristic

The Euler characteristic is a topological invariant computed from the mesh element counts. For a closed orientable surface it determines the genus g (number of handles), relating to the mesh's topological complexity.

χ = V − E + F

χ = 2 − 2g   (closed, orientable)

A sphere has χ=2, g=0. A torus has χ=0, g=1. MetroJS reports these values for each uploaded mesh.

🔵 Connected Components

BFS traversal over the face adjacency graph identifies all connected components. A mesh with multiple components may indicate floating geometry or separate parts, which can affect the Metro analysis if the reference surface is not fully connected.

BFS: O(V + E)

📐 Vertex Normals

Vertex normals are computed by averaging the face normals of all adjacent triangles, weighted by face area (implicitly via the cross-product magnitude). They are used for smooth shading in the renderer and for the sign computation in the Metro error mapping.

05 — Results & Discussion

Interpreting Metro Output

How to read the error metrics and heatmap produced by MetroJS, and what they reveal about mesh simplification quality.

📏

Hausdorff Distance

The one-sided Hausdorff distance H(A→B) is the maximum over all sample points on A of the distance to the nearest point on B. It captures the worst-case error. A symmetric metric H(A,B) = max(H(A→B), H(B→A)) can be obtained by running the algorithm in both directions.

📉

RMS Error

The Root Mean Square error averages squared distances across all sample points. It is more representative of overall mesh quality than Hausdorff, being less sensitive to isolated outliers caused by small mesh features or sampling noise.

🌡

Signed Heatmap

The diverging colour map — cool tones for contraction, warm tones for expansion — lets a user immediately identify which regions of the mesh were over-simplified (pulled inward) versus which expanded outward. This directionality is not captured by unsigned metrics alone.

Discussion Notes

The Metro algorithm was chosen as the reference method because it remains the standard for mesh-to-mesh comparison in computational geometry research. Its stochastic nature means results are approximate but converge quickly: at 10,000 samples the error is typically within 5% of the true value, and at 50,000 samples this reduces to under 1% for meshes with uniform feature distribution.

The BVH acceleration is critical for tractable computation in the browser. Without it, each of S Monte Carlo samples would require O(F) triangle queries against the target mesh, giving O(S·F) total cost. The BVH reduces this to O(S·log F), enabling 10k samples against a 10k-face mesh to complete in under 2 seconds on modern hardware.

Limitations of the current implementation include: the analysis is one-sided (A→B only), no adaptive sampling based on curvature, and the renderer uses a software rasterizer rather than WebGL, limiting performance for meshes with more than ~50k faces.
